autodock_vina {emily} R Documentation

Run Molecular Docking with AutoDock Vina

Description

This function performs protein-ligand molecular docking using AutoDock Vina on a Linux system. It handles preparation of receptor and ligand PDB files into PDBQT format, configuration of the docking box, execution of Vina, and extraction of the top-scoring pose.

Usage

autodock_vina(prot.pdb, ligand.pdb,
    complex.pdb = './complex.pdb',
    score.txt = './score.txt',
    center = NULL,
    size = 'c'(25, 25, 25),
    num.modes = 10,
    energy.range = 4,
    exhaustiveness = 8,
    cpu = 1,
    seed = NULL,
    mgltools.dir = '/opt/mgltools',
    autodock.vina.dir = '/opt/autodock_vina',
    make.cleanup = FALSE);

Arguments

prot.pdb

Character string. Path to the receptor protein PDB file.

ligand.pdb

Character string. Path to the ligand molecule PDB file.

complex.pdb

Character string. Output path for the top-scoring complex (PDB format). Default: './complex.pdb'

score.txt

Character string. Output path for the docking scores. Default: './score.txt'

center

Numeric vector of length 3. (x, y, z) coordinates for the center of the docking box in Angstroms. If NULL (default), the centroid of the protein is used. Providing explicit coordinates is strongly recommended for accurate docking1,7.

size

Numeric vector of length 3. Size of the docking box in Angstroms (x, y, z dimensions). Default: c(25, 25, 25)

num.modes

Integer. Maximum number of binding modes to generate. Default: 10

energy.range

Numeric. Energy range (in kcal/mol) above the best mode to output. Default: 4

exhaustiveness

Integer. Exhaustiveness of the global search. Higher values may improve results but increase computation time. Typical range: 8-329. Default: 8

cpu

Integer. Number of CPU cores to use. Default: 1

seed

Integer. Random seed. Use for reproducible results. Default: NULL (uses Vina default)

mgltools.dir

Character string. Path to the MGLTools directory. Default: '/opt/mgltools'

autodock.vina.dir

Character string. Path to the AutoDock Vina directory. Default: '/opt/autodock'

make.cleanup

Logical. Should temporary files be removed after completion? Default: FALSE

Details

The function performs the following steps: \enumerate{ \item Prepares the receptor PDB file into PDBQT format using `prepare_receptor4.py` from MGLTools. \item Prepares the ligand PDB file into PDBQT format using `prepare_ligand4.py` from MGLTools. \item Estimates the docking box center (if not provided) by calculating the centroid of all protein atoms. \item Writes a Vina configuration file. \item Executes AutoDock Vina. \item Parses the output to extract the top-scoring pose and saves it as a PDB file. \item Saves the docking scores to a text file. }

Authors

Who wrote it

Value

Invisible NULL. The function writes two output files: the top-scoring complex structure (`complexpdb`) and a text file containing docking scores (`scoretxt`).

Examples

#\dontrun{

# Using protein centroid as box center (not always recommended)
autodock_vina(
prot_pdb = "protein.pdb",
ligand_pdb = "ligand.pdb",
complex_pdb = "best_complex.pdb",
score_txt = "docking_scores.txt"
)
# Providing explicit box center coordinates (recommended)
autodock_vina(
prot_pdb = "protein.pdb",
ligand_pdb = "ligand.pdb",
center = c(10.5, 22.3, 18.7), # Coordinates based on known binding site
size = c(20, 20, 20),
exhaustiveness = 16,
cpu = 4
)

#}
